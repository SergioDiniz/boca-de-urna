Introdução:

Uma das grandes preocupações atuais para os profissionais da área de Segurança da Informação é quanto a veracidade dos dados apurados em uma votação utilizando as urnas eletrônicas.

Uma solução proposta por uma equipe de uma Startup foi a de criar um sistema de votação paralelo, onde os usuários, através de aplicativos móveis e aplicativos web podem informar de forma anônima qual foi sua opção.

Para tanto é necessário provar que uma aplicação deste tipo tem viabilidade técnica e para isto desejase criar um aplicativo que prove este conceito e para isto foi proposto a criação deste projeto, denominado de PoCSurvey.

O PocSurvey deverá utilizar as mesmas tecnologias e arquitetura que contemplarão o projeto final, sendo estas:

  • Arquitetura em Camadas para separar as responsabilidades de Cliente, Servidor e Dados;
  • Sockets;
  • XML para formatar as mensagens.

Escopo do Projeto:

O projeto deverá possuir três módulos distribuídos e interoperáveis e cada módulo deverá resolver um problema específico.

Os principais módulos deste sistema são:

  • UIModule;
  • ServerModule
  • DataModule.

UIModule:

Este módulo é responsável por prover uma interface de comunicação com o usuário final. Pode ser utilizado Console, Swing, SWT ou UIWeb.


ServerModule:

Este módulo é responsável por receber os dados, prover a persistência dos mesmos e disponibilizálos para solicitações. Não deve possui uma interface gráfica.


DataModule:

Este módulo é responsável por realizar a persistência de dados em um SGBD como PostgreSQL ou MySQL.


Funcionalidades:

Código Nome Descrição
F001 Registrar Usuário Um usuário anônimo pode acessar a aplicação e através de um formulário específico criar um usuário único a partir do seu email e gerando um token para este usuário.
  • token (calculado através do hash md5 do email)
F002 Acessar a aplicação Um usuário pode digitar apenas seu email e ter acesso a aplicação, desde que já este registrado no sistema.
F003 Votar Um usuário, após ser identificado como usuário válido poderá votar utilizando a opção numérica (número dos candidatos), desde que não tenha votado ainda, sendo necessário informalo que seu voto não é válido no caso de já ter votado antes.
F004 Apresentar Estatísticas Um usuário, mesmo que sem identificação, poderá ter acesso ao resumo dos dados de votação conforme os dados abaixo:
  • número do candidato
  • nome do candidato
  • quantidade de votos
  • proporção dos votos (em %)
F005 Sair da aplicação Um usuário, após ser identificado pela aplicação, poderá solicitar a sua saída da aplicação (logout).
